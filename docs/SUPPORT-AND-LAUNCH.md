# Support workflow and launch checklist

## Support intake

Request the plugin version, WordPress version, PHP version, browser, active theme, relevant plugins, exact reproduction steps, expected result, actual result, and non-sensitive screenshots. Never request passwords, license keys, payment-card data, or respondent records through ordinary email.

Classify requests as installation, builder/scoring, publishing/display, lead/email, license/billing, security/privacy, compatibility, feature request, or other. Security reports receive private handling and priority escalation.

## Launch checklist

- [ ] Final Free/Pro enforcement matches the pricing table.
- [ ] Fresh Free, fresh Pro, Free-to-Pro, Pro-to-Free, and 0.15.x upgrade paths pass.
- [ ] Scoring/profile, JSON, CSV, retention, export, erasure, and uninstall tests pass.
- [ ] PHP/WordPress and ecosystem compatibility evidence is recorded.
- [ ] Security, accessibility, translation, and coding-standard reviews are complete.
- [ ] Free ZIP installs without any licensing, account, checkout, opt-in, or telemetry screens.
- [ ] Direct AssessCraft Pro purchase, activation, update, refund, and account flows pass when Pro launches.
- [ ] Terms, privacy, license, and refund wording is approved by counsel.
- [ ] Documentation and support URLs are final.
- [ ] Production ZIP is reproducible and archive-tested.
- [ ] Database and file backups exist before release deployment.
- [ ] Rollback owner and decision threshold are documented.
- [ ] Version, changelog, tag, checksum, and release notes agree.
