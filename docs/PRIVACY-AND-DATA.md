# Privacy and data handling

AssessCraft processes assessment responses in the visitor’s browser and WordPress request flow. Merely completing an assessment does not store or transmit a result to the site owner.

Consultation-request storage is optional and disabled by default per assessment. When enabled and submitted with consent, the WordPress database can store contact details, assessment name, overall score, classification, matched profile, consent time, source URL, and submission time. Individual question answers are not stored in the consultation-request dashboard.

WordPress privacy integration provides:

- suggested privacy-policy content;
- personal-data export by email address;
- personal-data erasure by email address;
- configurable request retention;
- individual and bulk deletion;
- an explicit keep-or-delete uninstall choice.

Site owners remain responsible for selecting a lawful basis, providing an accurate privacy notice, collecting only necessary information, restricting access, configuring retention, securing backups, and handling data-subject requests.
